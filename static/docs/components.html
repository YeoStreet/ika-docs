Bundled Components

Ika provides several custom elements out of the box that plugs into the framework's core features to solve common front-end needs. These are all custom elements that extends the HTMLElement interface.

<ika-ab>

Binds child elements' attributes to states of the parent component. The binding can leap.

Attributes

data-*

The custom data attribute's name denotes the attribute of child elements to be bound. The custom data attribute's value denotes the state key of the parent component to bind to.

Note that dashes in the data attribute name is preserved, instead of the camel case querying the element's dataset property returns. Therefore, if you use an uppercase letter, it will be interpreted as having a dash before it - i.e. data-myAttr will bind the my-attr property of child elements.

Example

<ika-ab data-style="childStyleString">
   <span>Some text</span>
</ika-ab>

<ika-b>

Binds the text content or inner HTML of this element to a state of the parent component. The binding can leap.

Attributes

b

The state key to bind to in the parent component's state.

allow-html?

A boolean attribute that binds this element's innerHTML property to the state value instead of the default textContent.

Example

Hello, my name is <ika-b b="name"></ika-b>

<ika-bf>

Invokes an attached method of the parent component when a watched event is fired from child nodes. The binding can leap.

Attributes

data-*

The custom data attribute's name denotes the event type to watch for when the event bubbles up from a child node. The custom data attribute's value denotes which function to invoke in the parent component.

Example

<ika-bf data-click="clickHandler">
   <button>Submit form</span>
</ika-bf>

<ika-me>

Maps elements based on a state value of the parent component. The state value should conform to the IkaMeBindingValue type - learn more in the mapping data guide.

Attributes

mf

The map from key.

cn

The tag name of the component to map data to.

ika:is

The instance space name. The binding will be made to the parent component's state key ika-me:<instance name>

post?

Options are aside and remove. aside moves this element away from where it is initially to a special <ika:aside> element, while keeping the mapped child elements in place. remove removes this element after mapping child elements and keeping them in place.

Note that if remove is used, there will be no remapping of elements if the state value is changed, since this element, which binds to the state value, is removed.

<ika-import>

Import an Ika component from a URL. Learn more about the use of this element in the code splitting guide.

Attributes

src

The URL of the Ika component to be loaded onto the current page so that an element tag for that component can be rendered.

Example

<ika-import src="./resources/hello-world.html"></ika-import>
<hello-world></hello-world>

<ika-route>

A content container for the built-in router to enable navigation in your single page app (SPA). Alternatively, can be used to load in additional HTML content from a resource URL for code splitting practices. Learn more in the routing guide.

Attributes

key

A unique name to identify where to render content when routing is triggered.

src?

Resource URL for any initial content to be loaded, before any routing is triggered.

no-scroll-on-route?

By default, when new content is loaded into this element, the page scrolls to bring the top of this element into view. This boolean attribute if present will disable this behaviour.

Leaps in data bindings

In cases where a binding is made from an element nested inside multiple Ika components, you can specify a leap attribute on the element so that it will reach the "real" parent component.

Without the leap attribute, the binding will be made to the closest Ika component upwards in the DOM tree. A value of 1 for this attribute will allow the binding to skip one Ika component and be bound to the state of the next closest upwards in the DOM tree.
