Add Ika to your project

You can add Ika to your page by using the CDN link hosted on UNPKG. Since Ika is published as a module and does not define a window level variable, you can use an import statement in your own module like so: 

<script type="module">
   import { defineIkaComponent } from 'https://unpkg.com/ikajs';
   // Your code here...
</script>

Alternatively, you can install the package via NPM in your project, with yarn add ikajs or npm install ikajs.

Creating a component

To create a <hello-world> component, define the component's template first, and then register the component to make it available.

Ika puts all component templates into an element called ika:templates. To help Ika find and define the component, add the following to the page's HTML. 

<ika:templates>
   <template id="hello-world">
       Hello, world!
   </template>
</ika:templates>

Note the id attribute - this marks the element as the template for the component we're about to define.

Using a script tag, import Ika from UNPKG's CDN link and define the component, like so:

<script type="module">
   import { defineIkaComponent } from 'https://unpkg.com/ikajs';
   defineIkaComponent('hello-world')
</script>

Done! The component is now available to use as the <hello-world> element.

How this works, briefly

Ika makes use a set of standard Web APIs to create encapsulated, custom HTML elements. As with all custom elements, components created through Ika also follow naming rules, requiring at least one dash. The defineIkaComponent() function is at its root a wrapper over the customeElements.define() standard Web API.
